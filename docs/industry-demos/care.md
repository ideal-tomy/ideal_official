## 1. 楔（隙間）と証明する一文

### 楔

**現場で話せている・観察できている** ことと **帳票・申し送りとして残る記録** のあいだ。  
記録だけがシフト後に持ち帰る仕事になり、抜け漏れ確認でさらに時間が消える。

### 証明する一文

ケアの会話が、帰宅後の書き写しではなく、その場で構造化・下書きされ、業務として残る・回る。

## 3. Demo 詳細

### Demo① — 音声の構造化

- **Demo ID:** `care-voice-structured`
- **テーマ:** ケア記録のサンプル音声が、日付・対象・観察・対応などの項目に整う
- **体験してもらうこと:** サンプル音声 → 実行 → 構造化フィールドの確認
- **感じてほしいこと:** 「ゼロから書く」が「整った候補を直す」に変わる入口が分かる
- **主役役割:** 介護職・看護職・記録担当
- **クライマックス:** 話した内容が業務項目の一覧として揃う
- **ガイド手数:** 3
- **AIの使い方:** サンプル再生（マイク不要であることは明示済み想定）
- **既存資産:**
  - サイト内: `/ai-capability-gallery/voice-to-structured`
  - 事例: [`care-voice-records`](../../data/cases/care-voice-records.ts)
- **status:** `ready_asset`
- **ギャップ:** ハブ①としてのラベル。②③への「次は下書き／業務画面」導線
- **実装ステップ:**
  1. ハブカード①をギャラリー詳細へのリンク＋ケア文脈コピーで作成
  2. 完了後CTAを②へ
  3. 事例・CaseDemoCta と整合
- **受け入れ（Demo単体）:** 「音声が構造化記録になる」を自施設に置き換えられる

### Demo② — 申し送り・カルテ下書き

- **Demo ID:** `care-voice-karte`
- **テーマ:** 音声から申し送りカルテの下書きまで一気に出す
- **体験してもらうこと:** 音声入力〜カルテ／申し送り下書きの確認・修正感
- **感じてほしいこと:** 「構造化の先に、現場が使う帳票の形がある」
- **主役役割:** 現場・記録担当
- **クライマックス:** 下書きが帳票イメージで残り、直して使える
- **ガイド手数:** アプリ依存（ハブで開始手順を1画面分書く）
- **AIの使い方:** 外部デモ仕様に従う
- **既存資産:**
  - 公開: `https://lambent-smakager-7bcf0a.netlify.app/`（`EXTERNAL_DEMO_URLS.voiceKarte`）
  - portfolio: `voice-karte`（ready）
- **status:** `external_only`
- **ギャップ:** Netlify URLのブランド／更新手順。ideal ハブ②としての正式な位置づけコピー。WS内ソースの有無確認
- **実装ステップ:**
  1. ハブ②に voice-karte を紐づけ（portfolio 既存を流用）
  2. ①「項目になる」→②「帳票になる」の差分を1文で示す
  3. 必要なら ideal 配下または WS へのソース取り込みは別チケット
- **受け入れ（Demo単体）:** 「申し送り／カルテ下書きまで」を説明できる

### Demo③ — ケア業務オペ

- **Demo ID:** `care-ops`
- **テーマ:** ケア記録・オペが、業務アプリの画面として回る
- **体験してもらうこと:** 介護オペデモで記録〜確認の流れに触れる
- **感じてほしいこと:** 「下書きが、施設の運用画面に載る」
- **主役役割:** リーダー・管理者
- **クライマックス:** 業務画面上で記録まわりが回っていると分かる
- **ガイド手数:** アプリ依存
- **AIの使い方:** アプリ依存。①②がAI／下書き、③が運用の箱
- **既存資産:**
  - 公開: `https://kaigo-operation-demo.vercel.app/`（`EXTERNAL_DEMO_URLS.care`）
  - 現行TOP医療・福祉カードの `tryHref`
  - 事例 externalDemo も同URL
- **status:** `external_only`
- **ギャップ:** ①②との役割分担が訪問者に伝わりにくい（現状は③直リンクが本線）
- **実装ステップ:**
  1. TOP CTA をハブへ変更するまでの間、カード注記で「3体験のひとつ（オペ）」と分かるようにする方針を記載
  2. ハブでは③＝運用の箱、①②＝中の一手
  3. ログイン／開始摩擦があれば注記を追加
- **受け入れ（Demo単体）:** 「記録が業務アプリで回る」が①②の延長として想像できる
