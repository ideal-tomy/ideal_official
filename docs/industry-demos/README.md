# 4業界 × 3本建てデモ構成

最終更新: 2026-07-24

## 目的

ideal 公式サイトの業界カード本線を、**巨大なワークフロー1本**ではなく、**1テーマ×3本の体験デモ**で構成する。

訪問者は各デモで「効く一点」を実感し、ハブ上の組み合わせ文で **つながった業務導入** を想像する。  
本番開発はワークフロー前提のままだが、**デモ画面でワークフロー全体を再現しない**。

関連:

- 体験IA: [`../SITE_EXPERIENCE_REDESIGN.md`](../SITE_EXPERIENCE_REDESIGN.md)
- 業界カード: [`../../data/demo-first/top-page.ts`](../../data/demo-first/top-page.ts)
- 外部URL一覧: [`../../data/demo-first/portfolio.ts`](../../data/demo-first/portfolio.ts)

## 導線モデル

```text
TOP 業界カード
  → 業界ハブ（3テーマ）
       ├ Demo①（1テーマ）
       ├ Demo②（1テーマ）
       └ Demo③（1テーマ）
  → （頭の中で）組み合わせたワークフローを想像
  → ROI / 相談 / 要件整理
```

ギャラリー（`/ai-capability-gallery`）は **部品・試食**。業界本線の着地はハブまたは推奨 Demo①。

## 共通原則

| 原則 | 内容 |
|------|------|
| 1デモ1テーマ | クライマックスの質問／場面は1つ |
| 短時間 | ガイド3〜5手、数分で完走 |
| サンプル先行 | AI接続は効くデモだけ、ツアー後の任意 |
| 役割の偏り | 現場／管理／経営をデモごとに偏らせてよい |
| 組み合わせは文で | ハブに1行。WF画面は作らない |
| 3本厳守 | 4本目は関連部品か営業用隠しパラメータ |

### status 語彙（各業界 PLAN で使用）

| status | 意味 |
|--------|------|
| `ready_asset` | 体験資産があり、入口・コピー調整で公開可に近い |
| `needs_ux_split` | 同居・自由探索が強く、1テーマ導線の分離が必要 |
| `needs_build` | テーマ体験として新規／大幅補強が必要 |
| `external_only` | WS外ホスト。導線・期待値整備が主作業 |

## 業界索引

| 業界 | PLAN | 楔（隙間） | 組み合わせ1行 |
|------|------|------------|----------------|
| [製造](./manufacturing.md) | 先行・確定稿 | 文書 ↔ 現場判断 | 現場は①で動き、改定は②で揃え、設計変更は③で漏らさない |

## 新規業界を足すとき

1. [`_template.md`](./_template.md) を複製  
2. 楔・3テーマ・組み合わせ文を先に固定  
3. 既存資産と status を埋める  
4. 本 README の索引表に1行追加  

実装フェーズでは、各デモリポジトリのテーマ別要件定義書と Demo Definition（[`AI-Demo-Studio/docs/ai_demo_standard_definition_template.md`](../../../../AI-Demo-Studio/docs/ai_demo_standard_definition_template.md)）がこの PLAN を参照する。
