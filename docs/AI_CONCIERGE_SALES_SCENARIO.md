# AIコンシェルジュ — 営業デモ用シナリオ

最終更新: 2026-07-10  
関連: [`AI_CONCIERGE_REDESIGN.md`](./AI_CONCIERGE_REDESIGN.md) / [`AI_CONCIERGE_IMPLEMENTATION_PLAN.md`](./AI_CONCIERGE_IMPLEMENTATION_PLAN.md)

## 目的

商談・説明会で、

> AIコンシェルジュ型Webサイトを作れます

ではなく、

> うちのサイトで実際に使ってみてください

と言えるようにする。所要 **5〜8分**。

---

## 推奨ルート（1本）

```
TOP
 → デモギャラリー
 → 「写真 → 分類」デモを触る
 → AIコンシェルジュを開く（文脈オープン）
 → 状況・時期を選ぶ
 → 整理結果カードを見せる
 → 概算を出す
 → お問い合わせへ引き継ぐ（送信は任意）
```

代替: Cases「建設 × 現場写真整理」から入っても同じ型で説明できる。

---

## トークスクリプト（短版）

### 1. 入口（30秒）

「普通の制作会社サイトは、サービス説明を読んで問い合わせフォームです。  
ideal は **触ってから、自社に置き換えて、AIが整理する** ところまでサイト上で体験できます。」

→ TOP または `/ai-capability-gallery` を開く。

### 2. デモ体験（1〜2分）

「説明する前に、動くものを見ます。」

→ `/ai-capability-gallery/photo-to-classification`  
→ サンプルで分類の流れを見せる。

「これが『写真 → 分類』です。建設現場の写真整理などにそのまま応用できます。」

### 3. 文脈付きコンシェルジュ（1分）

右下の「相談」またはページ内の「自社でも使えるか相談する」を開く。

**ここで見せるポイント:**

- 第一声が「写真 → 分類」を理解していること
- よくあるチャットボットではなく、**今見ているデモの次の一手**になっていること

選択肢例:

- 自社業務で使えるか相談する
- 必要な機能を整理する
- 関連事例を見る
- 概算の感触をつかみたい

→ 「自社業務で使えるか相談する」を選ぶ。

### 4. 短いヒアリング（1分）

状況・時期を選ばせる（またはデモ操作者が選ぶ）。

「何を書けばいいか分からない人向けに、入口は選択式にしています。」

### 5. 整理結果（1分）

完了カードを見せる。

- 現在の状況 / 希望時期
- おすすめの方向性
- 想定機能タグ
- 関連デモ

「ここまでが、サイト上の簡易コンサルです。」

### 6. 概算（1分）

「概算費用を算出する」を押す。

**必ず言うこと:**

- 金額は AI が勝手に出していない
- 社内の価格ルール表に当てはめた **参考レンジ**
- 正式見積ではない（注記を指す）

### 7. 引き継ぎ（30秒）

「この内容で相談する」→ Contact に下書きが入ることを見せる。

「訪問者がここまで整理した状態で問い合わせるので、初回ヒアリングが短くなります。  
そしてこの導線自体を、御社サイトにも載せられます。」

---

## 伝えるべき3点

1. **文脈** — ページを理解して開く（右下チャットの置き換えではない）
2. **ハイブリッド** — 選択式入口 +（将来）自由入力・AI整理
3. **安全な概算** — 金額はルール、説明はAI（将来）/ テンプレート（現在）

---

## 相手業種別の差し替え例

| 相手 | デモ / Cases | 一言 |
|------|----------------|------|
| 工務店・建設 | 写真 → 分類 / 建設Cases | 現場写真の整理から |
| 介護 | 音声 → 構造化 / 介護Cases | 申し送り・記録から |
| 士業・M&A | 文書 → 抽出 / DD Cases | 資料読み込み負担から |
| 小売 | データ → 予測 | 需要・発注判断から |
| Web制作の発注検討 | Webサービスページ | 状況整理 → 概算 |
