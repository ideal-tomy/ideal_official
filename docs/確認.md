## 0. 使い方

### 0.1 前提

- 確認環境: `npm run dev`（ローカル）または本番相当 URL
- 端末: **PC とスマホの両方**（特に TOP・デモ詳細・FAB ナビ）
- 対象: 公開導線のみ（開発用 `/test-*` は削除済み）
- LAB / Philosophy / Research は「本線から迷わないか」を軽く見る程度

### 0.2 理想導線（合否の基準）

```
デモを見る → 自社業務に置き換える → AIコンシェルジュで相談
  → 要件整理・概算 → 問い合わせ
```

### 0.3 実施順序（厳守）

1. **フェーズ①** 掲載内容の理解しやすさ  
2. **フェーズ②** ユーザー導線  
3. **フェーズ③** demo 環境の満足度  

前段で NG が出ても後段は進めてよいが、**必ずメモを残してから進む**。改修優先度は末尾サマリーで整理する。

### 0.4 記録ルール

各項目の判定欄に次のいずれかを記入する。

| 記号 | 意味 |
|---|---|
| OK | 問題なし |
| NG | 要改善（「どこで・何が・なぜ分かりにくいか」を1行メモ） |
| — | 対象外 / 未実施 |

チェックボックス `- [ ]` は確認済みにしたら `- [x]` にする。
